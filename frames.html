Frames

Introduction

HTML frames were used to divide a web page into multiple sections where each section could load a different web page. Frames were widely used before the adoption of CSS layouts and flexbox/grid systems. However, frames are now deprecated in HTML5 because they cause usability and accessibility issues.

Although they are outdated, it's useful to understand them for maintaining older web projects.

Basic Structure of Frames

Frames are created using the <frameset> tag instead of <body>. Each frame is defined using the <frame> tag.

Example: Creating a Two-Column Layout

<frameset cols="50%,50%">
    <frame src="page1.html">
    <frame src="page2.html">
</frameset>

The cols="50%,50%" attribute divides the page into two vertical frames.

The src attribute loads different web pages inside the frames.

Attributes of the <frameset> Tag

AttributeDescription
colsDivides the page into columns (e.g., "30%, 70%")
rowsDivides the page into rows (e.g., "50%, 50%")
borderDefines the border width between frames
frameborderEnables (1) or disables (0) the border
noresizePrevents users from resizing frames

Creating Multiple Rows and Columns

Example: Creating a Layout with Two Rows

<frameset rows="30%, 70%">
    <frame src="header.html">
    <frame src="content.html">
</frameset>

This divides the window into two horizontal sections.

The first frame (header) takes 30% of the height.

The second frame (content) takes the remaining 70%.

Example: Creating a Complex Layout (Rows + Columns)

<frameset rows="20%,80%">
    <frame src="top.html" noresize>
    <frameset cols="30%,70%">
        <frame src="menu.html">
        <frame src="main.html">
    </frameset>
</frameset>

The first frame (header) takes 20% of the height.

The second part is divided into two vertical sections: menu (30%) and main content (70%).

Targeting Frames with Links

You can open links in a specific frame using the target attribute.

Example: Link That Opens in Another Frame

<frameset cols="30%,70%">
    <frame src="menu.html" name="menu">
    <frame src="main.html" name="content">
</frameset>
<!-- Inside menu.html -->
<a href="page2.html" target="content">Open in Right Frame</a>

Clicking the link loads page2.html in the right-side frame (content).

Advantages & Disadvantages of Frames

Advantages

  • Reusable Layout – One frame (e.g., menu) can stay fixed while others change.
  • Faster Loading – Only part of the page reloads instead of the whole page.

Disadvantages (Why Frames Are Deprecated)

  • SEO Issues – Search engines have difficulty indexing frame-based sites.
  • No Bookmarking – Users cannot bookmark individual pages within frames.
  • Navigation Problems – The browser’s back/forward buttons may not work correctly.
  • Not Mobile-Friendly – Frames do not adapt well to small screens.

Alternative to Frames: Using <iframe>

Unlike <frameset>, the <iframe> tag is still supported in HTML5. It allows embedding external pages inside a section of a webpage.

Example: Embedding a YouTube Video with <iframe>

<iframe width="560" height="315" src="https://www.youtube.com/embed/dQw4w9WgXcQ" allowfullscreen>
</iframe>

The video is embedded within a web page.

Responsive and better than traditional frames.

Conclusion

HTML frames were a useful feature in early web development for dividing a browser window into multiple sections, each capable of displaying a different document. However, due to significant limitations in usability, SEO, and mobile responsiveness, frames are now deprecated in HTML5.

For modern web design, <iframe> can still be used when embedding external content, but it should be used sparingly and with accessibility in mind. Developers are encouraged to use CSS Flexbox or Grid for layouts instead of frames.
