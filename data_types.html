Data Types

Introduction

JavaScript is a loosely typed language, meaning variables can hold values of any type without specifying their type explicitly. There are two main categories of data types:

  • Primitive Data Types
  • Non-Primitive (Reference) Data Types

Primitive Data Types

These are the most basic data types in JavaScript and are immutable (cannot be changed once created). They include:

Data TypeDescriptionExample
NumberRepresents numeric valueslet x = 10;
StringRepresents textual datalet name = "John";
BooleanRepresents true or falselet isActive = true;
NullRepresents an empty or unknown valuelet value = null;
UndefinedA variable that has been declared but not assigned a valuelet y;
SymbolRepresents a unique identifierlet sym = Symbol("id");
BigIntUsed for large integers beyond Number.MAX_SAFE_INTEGERlet bigNumber = 9007199254740991n;

Example

let num = 25;
let text = "Hello, JavaScript!";
let isAvailable = false;
console.log(typeof num); // Output: number
console.log(typeof text); // Output: string
console.log(typeof isAvailable); // Output: boolean

Non-Primitive (Reference) Data Types

These data types store references to memory locations. They include:

Data TypeDescription
ObjectA collection of key-value pairs
ArrayAn ordered list of values
FunctionA block of reusable code

Example

// Object
let person = { name: "John", age: 30 };
console.log(person.name); // Output: John

// Array
let colors = ["Red", "Green", "Blue"];
console.log(colors[0]); // Output: Red

// Function
function greet() {
    return "Hello, World!";
}
console.log(greet()); // Output: Hello, World!

Type Conversion in JavaScript

JavaScript allows implicit and explicit type conversion:

Implicit Conversion (Type Coercion)

console.log("5" + 5); // Output: "55" (String Concatenation)
console.log("5" - 3); // Output: 2 (Converted to Number)

Explicit Conversion

let str = "123";
let num = Number(str); // Converts string to number
console.log(typeof num); // Output: number

Mutable vs Immutable Objects

The terms mutable and immutable refer to whether an objectâ€™s state can be changed after it is created.

Mutable Objects

Mutable objects can be modified after their creation.

Changes to a mutable object affect the original object, rather than creating a new one.

Examples: Arrays, Objects

let arr = [1, 2, 3];
arr.push(4);  // Modifies the original array
console.log(arr); // Output: [1, 2, 3, 4]

Immutable Objects

Immutable objects cannot be modified after creation.

Any operation that tries to modify an immutable object creates a new object instead.

Examples: Strings, Numbers, Booleans

let str = "Hello";
str = str + " World";  // Creates a new string, does not modify the original one
console.log(str); // Output: "Hello World"

Key Differences

Feature Mutable Objects Immutable Objects
Can be changed? Yes No
Affects original object? Yes No, creates a new object
Examples Arrays, Objects Strings, Numbers, Booleans
Performance May be slower due to changes in memory Often faster due to optimization
