Introduction to JavaScript

Introduction

JavaScript is a lightweight, interpreted programming language that runs in the browser. It enables dynamic behavior on web pages such as interactivity, validations, DOM manipulation, animations, and more.

Why JavaScript?

  • Makes web pages interactive
  • Handles browser events and user interactions
  • Manipulates HTML and CSS dynamically
  • Performs calculations and validations
  • Communicates with servers using AJAX

Ways to Include JavaScript

JavaScript can be written in three ways:

  • Inline: Inside an HTML element using the onclick or similar attribute
  • Internal: Inside a <script> tag within the HTML file
  • External: Linking to a separate .js file

1. Inline JavaScript

JavaScript can be added directly inside an HTML tag using the onclick, onmouseover, or other event attributes.

Example:

<!DOCTYPE html>
<html>
<head>
  <title>Inline JavaScript</title>
</head>
<body>
  <button onclick="alert('Hello, JavaScript!')">Click Me</button>
</body>
</html>

Pros:

  • Simple and easy to use for small tasks.
  • No need for an external file.

Cons:

  • Difficult to maintain for large projects.
  • Mixing JavaScript with HTML is not a good practice.

2. Internal JavaScript

JavaScript can be placed inside a <script> tag within the <head> or <body> section of an HTML file.

Example:

<!DOCTYPE html>
<html>
<head>
  <title>Internal JavaScript</title>
  <script>
      function showMessage() {
          alert('Hello from Internal JavaScript!');
      }
  </script>
</head>
<body>
  <button onclick="showMessage()">Click Me</button>
</body>
</html>

Pros:

  • Keeps JavaScript separate from the HTML elements.
  • Easier to manage than inline JavaScript.

Cons:

  • Still mixes JavaScript within the HTML file, which may not be ideal for large applications.

3. External JavaScript

The best way to include JavaScript is by using an external file. This method enhances code organization and reusability.

Example:

<!-- HTML File -->
<!DOCTYPE html>
<html>
<head>
  <title>External JavaScript</title>
  <script src="script.js"></script>
</head>
<body>
  <button onclick="showMessage()">Click Me</button>
</body>
</html>
// script.js File
function showMessage() {
  alert('Hello from External JavaScript!');
}

Pros:

  • Improves code maintainability.
  • Allows multiple pages to share the same JavaScript file.
  • Keeps HTML and JavaScript separate for better readability.

Cons:

  • Requires an additional HTTP request to load the JavaScript file (can be minimized using caching).

Best Practices for Including JavaScript

  • Always use external JavaScript for better organization.
  • Place <script> tags before </body> to prevent blocking page loading.
  • Use event listeners instead of inline JavaScript to keep HTML clean.
  • Avoid global variables to prevent conflicts.

Summary

Method Pros Cons
Inline JavaScript Simple for small tasks Hard to maintain, not recommended
Internal JavaScript Separates logic from elements Still mixes HTML and JavaScript
External JavaScript Best for organization and reusability Requires an extra file

Assignment Solution: How to Add Scripts to a Page

<html>
<head>
    <title>All Script Types</title>

        // Internal JavaScript
        function showInternal() {
            alert('Hello from Internal JavaScript!');
        }
    </script>
    <script src="script.js"></script> <!-- External JavaScript -->

    <!-- Inline JavaScript -->
    <button onclick="alert('Hello from Inline JavaScript!')">Inline Button</button>

    <!-- Internal JavaScript Trigger -->

    <!-- External JavaScript Trigger -->

function showExternal() {
    alert('Hello from External JavaScript!');
}
