CSS Specificity

Introduction

CSS specificity determines which CSS rule applies when multiple rules target the same element. Specificity is calculated based on different types of selectors, the position in the stylesheet, and source order. Here's a breakdown of how it works:

1. Specificity Based on Selector Types

Selector Type Specificity Value Example
Universal Selector 0 * {}
Element Selector 1 h1 {}
Class Selector 10 .box {}
Attribute Selector 10 [type="text"] {}
Pseudo-class 10 :hover {}
ID Selector 100 #header {}
Inline Style 1000 style="color: red"
!important Overrides all color: red !important;

2. Specificity Based on Position (Source Order)

If two rules have the same specificity, the one that comes last in the stylesheet (or HTML) takes precedence.

3. Specificity Based on Source of the Style

Style Location Priority Level
External Stylesheet Lowest
Internal Style Higher
Inline Style Highest
!important Overrides all

4. Visual Hierarchy Representation

              !important
                  ↑
          Inline styles (1000)
                  ↑
          ID selectors (100)
                  ↑
   Class, attribute, pseudo-class (10)
                  ↑
       Element, pseudo-element (1)
                  ↑
          Universal selector (0)

If specificity values are equal, the last one wins (source order).

5. Examples

/* Specificity: 1 */
p {
  color: black;
}

/* Specificity: 10 */
.text {
  color: green;
}

/* Specificity: 100 */
#main {
  color: blue;
}

/* Specificity: 1000 */
<p style="color: red">Red text</p>

/* Overrides everything */
#main {
  color: blue !important;
}

Assignment

Create a single HTML file with a <div> element. Apply multiple styles from different sources:

  • An external style that sets the background to red.
  • An internal style that sets it to green.
  • An inline style that sets it to blue.
  • Add another internal rule using !important to override all.

Assignment Answer

<!DOCTYPE html>
<html>
<head>
  <style>
    /* Internal style */
    div {
      background-color: green;
    }

    .important-style {
      background-color: yellow !important;
    }
  </style>
  <link rel="stylesheet" href="style.css">
</head>
<body>

  <div class="important-style" style="background-color: blue;">Test Div</div>

</body>
</html>

External CSS (style.css)

div {
  background-color: red;
}

Expected Result: The div will have a yellow background due to the !important rule, even though the inline style sets it to blue.

Position

When two rules have the same specificity, the one that appears later in the code will override the earlier one.

Example

li {
  color: red;
  color: blue;
}

Result: color: blue; is applied.

Why? Because it's later in the same rule – this shows how position in code affects the final style.

Specificity (Selector Type-Based)

Specificity increases based on the type and complexity of the selector.

Example

li { color: blue; }                         /* Element selector - low specificity */
.first-class { color: red; }               /* Class selector - medium specificity */
li[draggable] { color: purple; }           /* Attribute selector - medium specificity */
#first-id { color: orange; }               /* ID selector - high specificity */

Result: If all rules apply to the same element, #first-id wins because ID selectors have higher specificity.

Type (Source of the CSS)

CSS can come from different sources, and they have different levels of importance by default.

Order of Precedence (Lowest to Highest):

  • External Stylesheets (<link>)
  • Internal Styles (<style>)
  • Inline Styles (style attribute)

Example

<link rel="stylesheet" href="style.css">  <!-- Lowest priority -->
<style>
  h1 { color: green; }                    <!-- Higher -->
</style>
<h1 style="color: blue;">Hello</h1>       <!-- Highest -->

Result: The inline style (style="color: blue;") will be applied unless something stronger like !important is used.

Importance (!important)

The !important keyword overrides all other declarations, regardless of specificity or position.

Example

color: red;
color: green !important;

Result: color: green will be applied due to !important.

Assignment

Create an HTML page that includes:

  • A paragraph (<p>) styled using:
    • An external CSS file
    • An internal <style> tag
    • An inline style
    • A class selector and an ID selector
    • A rule with !important

Assignment Solution

HTML

<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet" href="style.css">
  <style>
    #para {
      color: green;
    }
    .text {
      color: orange;
    }
    p {
      color: red !important;
    }
  </style>
</head>
<body>

  <p id="para" class="text" style="color: blue;">This is a test paragraph.</p>

</body>
</html>

External CSS (style.css)

p {
  color: black;
}

Result: Despite all styles, the color: red !important; from the internal style overrides all others.
